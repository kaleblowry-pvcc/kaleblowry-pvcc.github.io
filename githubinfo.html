All About GitHub

Q: What is a version control system? A: A Version Control System (VCS) is a tool that helps track and manage changes to files over time. It’s commonly used in software development to keep track of code history, making it easier to collaborate, review, and revert to previous versions if needed.
Q: What is Git? A: Git is a widely-used distributed version control system that tracks changes in files, primarily used in software development.
Q: What is GitHub? A: GitHub is a platform that hosts Git repositories, providing a place for developers to store, manage, and share their code online.
Q: What is the difference between Git and GitHub? A: The main difference between Git and GitHub is that Git is a version control system (VCS), while GitHub is a web-based platform that hosts Git repositories and offers additional tools to enhance collaboration and project management.
Q: Who started GitHub and how was it started? A: GitHub was founded in 2008 by Tom Preston-Werner, Chris Wanstrath, P. J. Hyett, and Scott Chacon.
Q: What company owns it now? A: GitHub is now owned by Microsoft, which acquired the company in 2018 for $7.5 billion.
Q: How much does a GitHub account cost? A: GitHub offers several pricing tiers to accommodate different needs:
Free:
Cost: $0
Includes unlimited public and private repositories, GitHub Actions, and limited storage (500MB for GitHub Packages and 2,000 GitHub Actions minutes per month).
GitHub Pro:
Cost: $4 per user/month
Adds advanced tools and features like more CI/CD minutes, additional storage (2GB for Packages and 3,000 Actions minutes per month), and insights into repositories to help developers improve productivity.
GitHub Team:
Cost: $4 per user/month (similar to Pro but for team management)
Designed for small teams, this plan includes collaboration tools like team-based permissions, a codespaces feature, and security controls.
GitHub Enterprise:
Cost: Starting at $21 per user/month
Provides large organizations with advanced security, compliance, and deployment options, including SAML single sign-on (SSO), advanced auditing, and enterprise-level support.
Q: What is Octocat? A: The Octocat is GitHub’s official mascot: a friendly, whimsical cartoon character that combines an octopus and a cat.

Git/Github Terms

Repository A repository is a central place where data, files, or other resources are stored and managed.
Commit A commit is a record of changes made to files in a repository.
Fork A fork is a personal copy of someone else's repository that is created in your own GitHub account.
Push A push refers to the action of uploading or sending your local commits (changes) from your local Git repository to a remote repository, such as the one hosted on GitHub.
Pull Requests A pull request (PR) is a feature that allows developers to propose changes to a codebase.
Workflows Workflows are automated processes defined in a repository that help manage tasks such as building, testing, and deploying code.
Issues Issues are a way to track tasks, enhancements, bugs, or any other types of requests related to a project.
Raw Button The Raw button appears when you view a file in a repository (such as a text file, script, or code file). Clicking the Raw button allows you to view the file's contents in plain, unformatted text without any additional GitHub interface elements, styling, or syntax highlighting.
Blame Button The Blame button provides a way to track the history of each line of code in a file. When you click the Blame button, it shows you who last modified each line of the file and when, allowing you to understand the evolution of the file over time and identify the person responsible for specific changes.
