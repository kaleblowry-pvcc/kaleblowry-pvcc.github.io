Software Engineering: Terms & Definitions

Term Definition
Software Engineering The branch of computer science that seeks principles to guide the development of large, complex software systems.
Integrated Development Environment (IDE) Combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package.
Association for Computing Machinery (ACM) Founded in 1947 as an international scientific and educational organization dedicated to advancing the arts, sciences, and applications of information technology.
Institute of Electrical and Electronics Engineers (IEEE) A global organization that promotes technological innovation and excellence in the fields of electrical, electronics, and computer engineering.
Software Life Cycle: Requirements Analysis The goal is to specify what services the proposed system will provide, identify conditions, and define how the outside world will interact with the system.
Software Life Cycle: Design Involves creating a plan for the construction of the proposed system.
Software Life Cycle: Implementation Involves the actual writing of programs, creation of data files, and development of databases.
Software Life Cycle: Testing Testing confirms the software product is compatible with requirements and ensures functionality by debugging and verification.
Modular Implementation The division of software into manageable units (modules) that each deal with a specific part of the overall software responsibility.
Unified Modeling Language (UML) A modern collection of tools developed with the object-oriented paradigm in mind.
Use Case A list of actions or steps a user takes to interact with a system or software to achieve a goal.
Use Case Diagram A visual representation of how different users (actors) interact with a system, illustrating the various functions and goals they can achieve.
Data Relationships: One-to-One Each entity is associated with one and only one other entity (e.g., a patient and a private room).
Data Relationships: One-to-Many One entity is associated with many others (e.g., a physician and their patients).
Data Relationships: Many-to-Many Many entities can associate with many others (e.g., consulting physicians and their patients).
Design Pattern A predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) A systematic approach to evaluating the quality and effectiveness of software engineering processes and products.
Software Testing: Glass Box Testing A testing technique that examines the internal structure, logic, and implementation of a program.
Software Testing: Black Box Testing A testing technique that evaluates system functionality without knowledge of its internal workings.
Software Testing: Beta Testing An opportunity for real users to use a product in a production environment to uncover bugs before general release.
Software Testing: Alpha Testing An early testing stage involving bug identification and issue resolution before public release.
Documentation: User Documentation Explains the features of the software and describes how to use them.
Documentation: System Documentation Describes the softwareâ€™s internal composition to facilitate later maintenance.
Documentation: Technical Documentation Describes how a software system should be installed and serviced, including updates and problem reporting.
Intellectual Property Law: Software License The rights of the developer, formally expressed in legal terms.
Intellectual Property Law: Software Disclaimer States the limitations of liability in the software to avoid litigation.
Intellectual Property Law: Defenses Against Liability Applying sound engineering principles, maintaining care compatible with application, and producing records are the best defenses against liability claims.
